# Assignment 2: Shakespeare-Aware RAG System

## CSCI433/933 Machine Learning Algorithms and Applications

A Retrieval-Augmented Generation (RAG) system for answering questions about
Shakespeare's _Hamlet_, _Macbeth_, and _Romeo and Juliet_.

---

## Requirements

- Python 3.9+
- [Ollama](https://ollama.com/download) installed and running
- `llama3.2` model pulled via Ollama

---

## Setup

### 1. Install Python dependencies

```bash
pip install -r requirements.txt
```

### 2. Install Ollama and pull the model

Download Ollama from https://ollama.com/download, then:

```bash
ollama pull llama3.2
```

Ollama starts automatically as a background service after installation.

---

## Running the System

All scripts must be run from the `src/` directory:

```bash
cd src/
```

### Step 1 — Preprocess the dataset

Converts raw scene JSONL files to flat JSON records in `data/processed/`.

```bash
python preprocess.py
```

### Step 2 — Build the embedding index

Encodes all 73 scene chunks and caches embeddings to `data/index/`.
Only needs to run once; subsequent runs load from cache automatically.

```bash
python build_index.py
```

### Step 3 — Run the RAG chatbot

Interactive command-line chatbot. Type a question and press Enter.
Type `quit` to exit.

```bash
python rag_chatbot.py
```

### Step 4 — Run the evaluation

Runs all 10 evaluation questions through both the baseline and RAG systems.
Saves results to `results/evaluation_results.csv`.

```bash
python evaluate.py
```

### Optional — Run the baseline only

```bash
python baseline.py
```

## System Overview

| Component        | Choice                     | Justification                       |
| ---------------- | -------------------------- | ----------------------------------- |
| Chunking         | Scene-level (73 chunks)    | Preserves conversational context    |
| Embedding model  | all-MiniLM-L6-v2           | Lightweight, fast, no GPU needed    |
| Retrieval        | Cosine similarity (top-3)  | Simple, effective, no external deps |
| Generation model | llama3.2 (3.2B) via Ollama | Runs locally on CPU, no API key     |
| Baseline         | Prompt-only (same model)   | Isolates the effect of retrieval    |

---

## Notes

- Embedding index is cached after first build — `rag_chatbot.py` and `evaluate.py`
  load from cache on subsequent runs.
- Ollama must be running when using the chatbot or evaluation scripts.
- All dependencies are CPU-friendly; no GPU is required.
